# 📋 Engenharia e Elicitação de Requisitos

Repositório dedicado aos estudos, atividades e projetos desenvolvidos na
disciplina **Engenharia e Elicitação de Requisitos**.

Essa disciplina é fundamental para o desenvolvimento de software, pois
define **o que realmente precisa ser construído em um sistema**.

------------------------------------------------------------------------

# 🎯 Objetivo da Disciplina

A disciplina tem como objetivo ensinar como:

-   identificar necessidades dos usuários
-   levantar requisitos de software
-   documentar funcionalidades do sistema
-   validar requisitos com stakeholders
-   reduzir erros nas fases iniciais do desenvolvimento

------------------------------------------------------------------------

# 🧠 Conteúdos Estudados

Principais tópicos abordados:

-   Engenharia de Requisitos
-   Elicitação de Requisitos
-   Requisitos Funcionais
-   Requisitos Não Funcionais
-   Casos de Uso
-   Histórias de Usuário
-   Técnicas de levantamento de requisitos
-   Documentação de software

------------------------------------------------------------------------

# 🛠 Técnicas de Elicitação

Algumas técnicas utilizadas para levantamento de requisitos:

-   Entrevistas com usuários
-   Brainstorming
-   Workshops
-   Observação de usuários
-   Prototipação
-   Questionários

Essas técnicas ajudam a entender **o problema que o sistema precisa
resolver**.


------------------------------------------------------------------------

# 📄 Artefatos Produzidos

Durante a disciplina podem ser produzidos diversos documentos
importantes:

-   Documento de Requisitos de Software (DRS)
-   Diagramas de Caso de Uso
-   Histórias de Usuário
-   Protótipos
-   Especificações de requisitos
